Recommended tools

Project IDX

Good when you want a fuller cloud IDE feel, templates, and a space that behaves more like a real project environment.

  • Cloud workspaces
  • Good for structured app projects
  • Strong fit for Chromebooks

CodePen

Best for fast frontend experiments, tiny demos, UI sketches, and code you want to see instantly.

  • Low friction
  • Great for HTML/CSS/JS play
  • Less ideal for multi-file apps

StackBlitz

Excellent when you need a more serious browser environment with Node, npm, Vite, and framework workflows.

  • WebContainers
  • Good for React/Vite projects
  • Feels closer to local dev

Quick choosing rule

Use CodePen for toy UI work, StackBlitz for real browser app builds, and IDX when you want a broader cloud IDE setup.

Tool budgeting worksheet

Free-only month

  • Use CodePen or StackBlitz.
  • Use free model tiers lightly.
  • Focus on finishing small things, not browsing tool menus.

One paid tool month

  • Buy the thing you live in all day.
  • Usually that means an AI editor or assistant.
  • Cancel anything that is just ambient guilt.

Two-tool setup

  • One editor assistant.
  • One generation or prototyping tool.
  • Keep roles distinct so overlap stays minimal.

Decision prompts

  • What task keeps slowing me down?
  • Does this tool remove that friction?
  • Will I actually use it three times a week?

Chromebook Linux quick start

1. Enable Linux

Open Settings → Developers → Linux development environment, click Turn On, allocate space, and finish install.

2. Update packages

sudo apt update && sudo apt dist-upgrade -y

3. Install core tools

sudo apt install -y git curl build-essential

4. Install NodeJS 20

curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash - && sudo apt install -y nodejs

5. Use the file bridge

Share a Chromebook folder with Linux, then access it under:

/mnt/chromeos/MyFiles/
cp -r "/mnt/chromeos/MyFiles/welcome to vibe coding" ~/vibe-coding-kit

Dexter's Tool Verdict

You answered five questions. Here are the three real options, then my pick. I don't do "it depends."

Project IDX (Google)

What: Browser-based cloud IDE built on VS Code. Runs on any device.

Excels: Zero setup, Chromebook-native, phone-accessible, team collaboration, generous free tier.

Weak: Needs internet, slower than local for big projects, smaller ecosystem.

Price: Free tier; paid from ~$10/mo; Google Cloud + Gemini billed separately.

Claude Code (Anthropic)

What: AI-first coding environment with Claude integrated directly.

Excels: Seamless AI pair-programming, mobile-friendly, privacy-conscious, predictable per-token pricing.

Weak: Lighter on deploy/DevOps, no built-in hosting, limited multi-user collab.

Price: Limited free tier; ~$20/mo standard, ~$200/mo pro; per-token option.

OpenAI Codex (ChatGPT)

What: Code generation + assistant via ChatGPT/API. Not a full IDE.

Excels: Strong code reasoning, huge community, mature API, clear explanations.

Weak: No dedicated mobile IDE, needs a separate editor, unpredictable token cost, data-privacy concerns.

Price: Free tier limited; Plus $20/mo; API usage-based and can spike.
